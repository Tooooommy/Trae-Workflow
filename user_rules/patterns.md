# 架构模式

## API 响应格式

```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "meta": { "total": 100, "page": 1, "limit": 20 }
}
```

## 仓储模式

| 操作     | 说明                 |
| -------- | -------------------- |
| findAll  | 获取所有记录         |
| findById | 根据 ID 获取单条记录 |
| create   | 创建新记录           |
| update   | 更新记录             |
| delete   | 删除记录             |

**原则**：业务逻辑依赖抽象接口，不依赖存储机制。
