# 架构模式

> 定义架构模式。

---

## API 响应格式

具有成功指示器、数据负载、错误消息和分页元数据的一致信封。

```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "meta": {
    "total": 100,
    "page": 1,
    "limit": 20
  }
}
```

---

## 仓储模式

将数据访问封装在标准接口后面：

| 操作 | 说明 |
|------|------|
| findAll | 获取所有记录 |
| findById | 根据 ID 获取单条记录 |
| create | 创建新记录 |
| update | 更新记录 |
| delete | 删除记录 |

**原则**：业务逻辑依赖于抽象接口，而不是存储机制。

## 骨架项目

实现新功能时：

1. 搜索经过实战检验的骨架项目
2. 使用并行智能体评估选项：
   - 安全性评估
   - 可扩展性分析
   - 相关性评分
3. 克隆最佳匹配作为基础
4. 在已验证的结构内迭代
