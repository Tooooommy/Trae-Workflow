# 代码规范

> 定义代码风格配置。

---

## 编码原则

| 原则     | 说明                                                          |
| -------- | ------------------------------------------------------------- |
| 不可变性 | 总是创建新对象，永不修改。返回带有更改的新副本                |
| 文件组织 | 许多小文件优于少数大文件。通常 200-400 行，最多 800 行        |
| 错误处理 | 在每个层级处理错误。UI 提供用户友好消息，服务端记录详细上下文 |
| 输入验证 | 在系统边界验证所有用户输入。使用基于模式的验证。快速失败      |

---

## 代码格式

| 配置项 | 值                              |
| ------ | ------------------------------- |
| 缩进   | 2 空格                          |
| 行长度 | ≤ 100 字符                      |
| 引号   | 单引号（'）除非字符串包含单引号 |
| 分号   | 语句末尾必须加分号              |
| 大括号 | K&R 风格（左大括号不换行）      |

---

## 文件命名

| 文件类型 | 命名规则             | 示例             |
| -------- | -------------------- | ---------------- |
| 组件文件 | PascalCase           | UserProfile.tsx  |
| 工具文件 | kebab-case           | user-utils.ts    |
| 类型文件 | kebab-case.types     | user.types.ts    |
| 常量文件 | kebab-case.constants | api.constants.ts |

---

## 代码质量检查清单

- [ ] 函数小巧（<50 行），文件专注（<800 行）
- [ ] 没有深层嵌套（>4 层）
- [ ] 适当的错误处理，没有硬编码的值
- [ ] 可读性强、命名良好的标识符
