# 代码规范

## 编码原则

| 原则     | 说明                                          |
| -------- | --------------------------------------------- |
| 不可变性 | 创建新对象，永不修改现有对象                  |
| 文件组织 | 小文件优于大文件，通常 200-400 行，最多 800 行 |
| 错误处理 | 每层处理错误，UI 友好消息，服务端详细日志      |
| 输入验证 | 系统边界验证，快速失败                        |

## 代码格式

| 配置项 | 值                   |
| ------ | -------------------- |
| 缩进   | 2 空格               |
| 行长度 | ≤ 100 字符           |
| 引号   | 单引号               |
| 分号   | 句末加分号           |
| 大括号 | K&R 风格             |

## 文件命名

| 文件类型 | 命名规则         | 示例             |
| -------- | ---------------- | ---------------- |
| 组件     | PascalCase       | UserProfile.tsx  |
| 工具     | kebab-case       | user-utils.ts    |
| 类型     | kebab-case.types | user.types.ts    |

## 检查清单

- [ ] 函数 < 50 行，文件 < 800 行
- [ ] 无深层嵌套（> 4 层）
- [ ] 错误处理完善
- [ ] 命名清晰
