# 测试规范

## 测试类型

| 类型       | 范围                 |
| ---------- | -------------------- |
| 单元测试   | 单个函数、组件       |
| 集成测试   | API、数据库操作      |
| 端到端测试 | 关键用户流程         |

## 覆盖率要求

- **最低**：80%
- **目标**：90%+

## TDD 工作流

```
1. RED   → 先写测试，测试失败
2. GREEN → 最小实现，测试通过
3. IMPROVE → 重构，验证覆盖率
```

## 测试策略

| 策略   | 说明                   |
| ------ | ---------------------- |
| 测试优先 | 先写测试，再写代码   |
| 隔离性 | 每个测试独立运行       |
| 可读性 | 名称清晰描述测试内容   |
| 快速   | 单元测试 < 50ms       |

## 测试失败排查

1. 使用 **tdd-guide** 智能体
2. 检查测试隔离性
3. 验证模拟正确
4. 修复实现，非必要不修测试
