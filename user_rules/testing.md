# 测试规范

> 定义测试要求。

---

## 测试类型

| 类型 | 范围 |
|------|------|
| 单元测试 | 单个函数、工具、组件 |
| 集成测试 | API 端点、数据库操作 |
| 端到端测试 | 关键用户流程 |

---

## 覆盖率要求

- **最低要求**：80%
- **推荐目标**：90%+

---

## TDD 工作流

```
1. 先写测试（RED）  → 测试应该失败
2. 编写最小实现（GREEN） → 测试应该通过
3. 重构（IMPROVE） → 验证覆盖率 80%+
```

---

## 测试策略

| 策略 | 说明 |
|------|------|
| 测试优先 | 先写测试，再写代码 |
| 隔离性 | 每个测试独立运行 |
| 可读性 | 测试名称清晰描述测试内容 |
| 快速 | 单元测试每个 < 50ms |

---

## 测试失败排查

1. 使用 **tdd-guide** 智能体
2. 检查测试隔离性
3. 验证模拟是否正确
4. 修复实现，而不是测试（除非测试有误）
