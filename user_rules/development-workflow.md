# 开发工作流

## 4 步流程

```
1. 规划 → planner 智能体
2. TDD  → tdd-guide 智能体
3. 审查 → code-reviewer 智能体
4. 提交 → 约定式提交
```

## 详细流程

### 1. 规划

- planner 智能体创建计划
- 识别依赖和风险
- 分解阶段

### 2. TDD

- tdd-guide 智能体
- RED → 测试失败
- GREEN → 最小实现
- IMPROVE → 重构，验证覆盖率

### 3. 审查

- code-reviewer 智能体
- 修复 CRITICAL/HIGH
- 尽量修复 MEDIUM

### 4. 提交

- 详细提交信息
- 约定式格式
- 全面 PR 摘要

## 研究与复用

1. GitHub 代码搜索
2. 包注册表搜索
3. 优先采用经过验证的方法
