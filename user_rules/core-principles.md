# 核心原则

> 此文件定义项目的核心原则，优先级最高。

---

## 五条基本原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **智能体优先** | 将领域任务委托给专业智能体 |
| 2 | **测试驱动** | 先写测试再实现，要求 80%+ 覆盖率 |
| 3 | **安全第一** | 绝不妥协安全；验证所有输入 |
| 4 | **不可变性** | 总是创建新对象，永不修改现有对象 |
| 5 | **先规划后执行** | 在编写代码前规划复杂功能 |

---

## 原则详解

### 1. 智能体优先

复杂任务应委托给专业智能体处理，而非手动完成。详见 [agents.md](agents.md)。

### 2. 测试驱动

所有代码必须先写测试，确保覆盖率 ≥ 80%。详见 [testing.md](testing.md)。

### 3. 安全第一

安全不可妥协，所有输入必须验证，敏感数据必须保护。详见 [security.md](security.md)。

### 4. 不可变性

总是创建新对象，永不修改现有对象。返回带有更改的新副本。

```
WRONG:  modify(original, field, value) → 原地修改
CORRECT: update(original, field, value) → 返回新副本
```

### 5. 先规划后执行

复杂功能必须先规划再实现。详见 [development-workflow.md](development-workflow.md)。
