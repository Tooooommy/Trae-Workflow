# Skill Creator

用于创建新 Skills 并迭代改进的技能。

## 何时激活

当用户要求以下任一操作时激活本技能：

- 创建新的 Skill
- 编辑或优化现有 Skill
- 运行评估测试 Skill
- 基准测试 Skill 性能
- 优化 Skill 描述以提高触发准确性
- 验证 Skill 结构或格式

## 工作流程

```
1. 决定 → 明确目标
2. 编写 → 创建 SKILL.md
3. 测试 → 验证 Skill 有效
4. 迭代 → 根据反馈改进
```

## 创建 Skill

### 1. 确定意图

理解用户的需求：

1. 这个 Skill 应该让 AI 做什么？
2. 什么时候触发这个 Skill？
3. 期望的输出是什么？

### 2. 设计 Skill 结构

```
skill-name/
├── SKILL.md      # 必需：指令 + 元数据
├── scripts/      # 可选：可执行代码
├── references/   # 可选：文档资料
└── assets/       # 可选：模板、资源
```

### 3. 编写 SKILL.md

````markdown
---
name: skill-name
description: 简洁描述 Skill 做什么以及何时使用。
---

# Skill 标题

详细描述...

## 何时激活

- 场景1
- 场景2

## 最佳实践

- 要点1
- 要点2

## 代码示例

```代码示例

```
````

````

### Description 编写指南

Description 是主要触发机制，让它有推动性：

**避免**：模糊描述
```markdown
"如何构建仪表板"
````

**推荐**：明确触发上下文

```markdown
"如何构建仪表板。当用户提到仪表板、数据可视化、内部指标或想要显示任何类型公司数据时，确保使用此 Skill，即使他们没有没有明确要求'仪表板'。"
```

### Skill 编写原则

| 原则 | 说明                        |
| ---- | --------------------------- |
| 简洁 | 保持 SKILL.md 在 500 行以下 |
| 清晰 | 从 SKILL.md 清楚引用文件    |
| 安全 | 不得包含恶意软件或误导内容  |
| 实用 | 解释为什么重要，不过度限制  |

### 安全要求

Skills 不得包含：

- 恶意软件或利用代码
- 旨在促进未授权访问的内容
- 误导性 Skills
- 让用户感到意外的内容

## 改进 Skill

### 评估流程

1. **收集反馈** - 从测试用例中获取用户评估
2. **分析模式** - 识别重复问题和改进空间
3. **优化描述** - 提高触发准确性
4. **精简内容** - 删除没有发挥作用的内容

### 优化描述

```bash
# 检查描述是否清晰
python -m scripts.improve_description <skill-path>
```

## Skill 评估

### 测试用例设计

为 Skill 设计 2-3 个现实的测试提示：

```markdown
测试提示 1: "用户实际会说的话..."
测试提示 2: "另一个真实场景..."
测试提示 3: "边界情况..."
```

### 评估标准

| 类型     | 可验证性 | 评估方式 |
| -------- | -------- | -------- |
| 转换类   | 客观     | 文件对比 |
| 生成类   | 主观     | 人工审查 |
| 工作流类 | 可测试   | 步骤验证 |
