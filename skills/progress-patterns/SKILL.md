# 项目进度追踪与优化模式

跟踪记录项目进度，生成进度文件，提供项目功能优化建议。

## 何时激活

- 创建新项目时初始化进度文件
- 阶段开始或完成时更新进度
- 每日检查并更新项目状态
- 发现阻塞事项时记录
- 项目完成时生成总结报告
- 需要功能优化建议时

## 核心职责

1. **进度追踪** - 跟踪记录各阶段进度
2. **文件管理** - 生成和维护 progress.md
3. **状态更新** - 及时更新任务状态
4. **阻塞管理** - 记录和跟踪阻塞事项
5. **优化建议** - 提供功能优化建议

## 输入要求

### 项目基本信息

```markdown
## 项目信息
- 项目名称：xxx
- 创建时间：2024-01-01
- 预计完成：2024-01-31
- 当前状态：进行中
```

### 阶段状态

```markdown
## 阶段进度更新
- 阶段：阶段 3：开发
- 模块：frontend-patterns
- 状态：进行中 -> 已完成
- 完成度：60% -> 100%
```

## 功能优化建议

### 性能优化

```markdown
## 性能优化建议

### 前端优化
1. **代码分割** - 使用 React.lazy 进行路由级代码分割
2. **图片优化** - 使用 WebP 格式，懒加载图片
3. **缓存策略** - 配置合理的 Cache-Control

### 后端优化
1. **数据库索引** - 为常用查询字段添加索引
2. **Redis 缓存** - 热点数据缓存至 Redis
3. **连接池** - 配置合理的数据库连接池大小

### API 优化
1. **分页** - 大数据量列表使用分页
2. **字段过滤** - 支持 select 指定返回字段
```

### 代码质量优化

```markdown
## 代码质量优化建议

### 重构建议
1. **提取公共组件** - 将重复的 UI 逻辑提取为公共组件
2. **服务拆分** - 将大文件拆分为小模块
3. **类型优化** - 使用更精确的 TypeScript 类型

### 最佳实践
1. **错误处理** - 统一错误处理方式
2. **日志规范** - 结构化日志输出
3. **测试覆盖** - 补充关键路径测试
```

### 架构优化

```markdown
## 架构优化建议

### 可扩展性
1. **插件化** - 将功能模块插件化
2. **配置外置** - 将硬编码配置外置到配置文件
3. **服务拆分** - 按业务边界拆分服务

### 可维护性
1. **文档完善** - 补充 API 文档和代码注释
2. **监控告警** - 完善系统监控和告警
3. **灰度发布** - 支持灰度发布策略
```

## 跟踪规则

1. **每个阶段开始时** - 更新对应阶段的进度
2. **每日更新** - 记录每日进度变化
3. **状态变更** - 立即更新任务状态
4. **阻塞事项** - 发现时立即记录并标记

## 进度更新时机

| 时机 | 操作 |
| ---- | ---- |
| 阶段开始 | 创建阶段进度记录 |
| 任务完成 | 更新任务状态为 ✅ |
| 任务开始 | 更新任务状态为 🔄 |
| 发现阻塞 | 记录至阻塞事项 |
| 每日检查 | 更新最近更新时间 |
| 项目完成 | 生成项目总结报告 |

## 质量门禁

| 检查项 | 阈值 |
| ------ | ---- |
| 进度更新 | 24h 内 |
| 阻塞记录 | 立即 |
| 优化建议 | 按需 |
